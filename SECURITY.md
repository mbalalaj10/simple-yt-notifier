# Security Policy
The Simple YT Notifier maintainer(s) strive to keep the project free of any critial security issues that potentially may arise.

## Supported Versions
We provide security updates to the **latest 5 versions** of Simple YT Notifier.

## Reporting a Vulnerability
Please report any security vulnerabilities through the **GitHub Private Vulnerability Reports** feature.
This feature allows you to report security vulnerabilities directly without exposing them to the public.
All reports are taken seriously, and will be responded to as quickly as possible.

**[Click here to report a vulnerability privately](https://github.com/mbalalaj10/simple-yt-notifier/security/advisories/new)**

### Disclosure Process & Timelines
* **Vulnerability Disclosure:** Once a report is received, it will be kept private until a fix is released.
* **Acknowledgement:** We will acknowledge your report within **72 hours**.
* **Remediation:** We aim to provide a patch and **publicly disclose** the fix within **90 days**.

## Security Practices
This project utilizes the following security measures:
* **OpenSSF Scorecard** for supply chain security.
* **GitHub CodeQL** for Static Application Security Testing (SAST).
* **Dependabot** for dependency management.
